# 耗时操作的乐观更新、Shimmer 和状态反馈

## 分支

```text
feature/optimistic-ui-feedback
```

## 本次范围

这个分支只处理用户已经触发操作后的“体感等待”问题，不新增业务功能、不修改数据库结构。

重点覆盖：

- 搜索提交：提交后按钮显示处理中状态，避免重复点击。
- 收藏 / 取消收藏：保留本地即时切换，失败后回滚，并增加处理中反馈。
- 好友申请：发送后先显示正在发送给对应用户，成功后显示已发送，失败后恢复可重试。
- 报名 / 取消报名：提交后显示明确处理中状态。
- 创建 / 编辑组队：发布或保存时显示处理中状态。
- 取消组队：站内确认弹窗和确认按钮都有处理中状态。
- 消息发送：发送按钮显示处理中状态。

## 体验原则

- 用户点击后必须立刻看到反馈。
- 不把失败的操作伪装成成功。
- 按钮处理中时禁用重复点击。
- 移动端按钮文字不换行，状态提示不遮挡底部导航。
- 页面级加载继续使用已有 loading / shimmer，不在本分支重复扩展。

## 验收

- 搜索、收藏、好友申请、报名、取消、发布和发送消息都有可见处理中状态。
- 数据库同步失败时保留原有错误提示或回滚逻辑。
- 同一按钮在处理中不会被重复点击。
- 移动端按钮和提示不会挤压换行。
