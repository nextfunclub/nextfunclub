# v1.2 功能实现清单

## 当前阶段目标

v1.2 仍处于进行中阶段，核心目标是把“活动信息”和“用户组队”拆清楚，同时补齐社交、评论和举报相关的基础能力。

当前主线：

```text
用户发现活动信息 → 基于活动发起组队 → 好友或陌生人加入组队 → 通过评论 / 私聊 / 通知完成活动前沟通
```

## 分支规则

所有功能从 `dev` 拉独立 feature 分支：

```bash
git switch dev
git pull origin dev
git switch -c feature/功能名
```

完成后 PR 到：

```text
feature/* → dev
```

不要把评论、举报、好友入口、私聊等功能合并到同一个分支里。每个分支只解决一个明确问题，方便测试、回滚和团队协作。

## P0：v1.2 必须收口的核心体验

### 1. 活动信息位与组队模型收口

建议分支：

```text
feature/public-event-team-model-plan
```

小功能：

- [x] 区分“活动信息”和“组队”
- [x] 活动信息卡片不展示报名人数和名额状态
- [x] 活动信息页只作为信息详情页，不直接报名
- [x] 用户可以基于活动信息发起组队
- [x] 组队可以被报名、审核、取消报名
- [x] 已有 API / 爬虫导入的旧活动兼容为活动信息
- [x] 用户自发的组队不允许再被复制成另一个组队
- [x] 活动发现页保持原有卡片排布，只调整语义和操作文案

验收标准：

- 活动发现页展示的是活动信息和正在组队的车队
- 活动信息卡片不出现 `0/99`、名额、已报名等人数信息
- 活动信息详情页可以进入“发起组队”流程
- 组队详情页保留报名、审核、取消报名能力
- 旧导入活动在新模型下可以正常展示和跳转
