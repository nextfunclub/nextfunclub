# Next Fun Club v1.2 更新公告草稿

状态：进行中，尚未发布到 `main`

v1.2 重点把“活动”和“用户组队”拆成更清楚的两条路径：活动发现页用于看真实活动，组队大厅用于看正在约人的组局。与此同时，本版补齐了好友入口、发起人私聊、评论回复、举报处理、运营数据和游客访问体验，让产品从“能用”进一步走向“更好用、更可运营”。

## 主要亮点

1. 活动和组队逻辑彻底拆清楚。
2. 组队大厅、组局发布页和游客访问体验更顺。
3. 好友、评论、举报和通知补齐活动前沟通与安全反馈。
4. 运营数据看台上线，管理员可以看到产品和活动的真实表现。
5. 微信内置浏览器登录引导和品牌 loading 动效提升细节体验。

## 用户可感知更新

1. 活动发现页继续保留原来的卡片浏览体验，但活动和组局会用更清楚的标签和操作区分。
2. 从公开来源导入的活动会作为“活动”展示，不再强行显示报名人数、名额和 `0/99` 这类误导信息。
3. 用户可以先查看一个活动，再基于这个活动发起自己的组队。
4. 已经存在的组局不再继续派生新的组局，避免“组队套组队”的层级混乱。
5. 组队大厅集中展示正在约人的组局，适合查看自己、好友和平台上的组队机会。
6. 组队大厅增加更清楚的筛选、空状态和移动端布局，用户不会一进页面就被大量说明挡住。
7. “我来组局”发布页改为组队语义，不再使用“创建活动 / 新增活动”这类容易混淆的文案。
8. 组局发布页默认不限制人数，只有用户主动选择人数限制时才需要填写上限。
9. 从链接导入活动默认收起，需要时再展开并解析链接，页面更干净。
10. 游客访问 `/` 会进入首页，游客也可以先浏览首页和组队大厅内容，不会一上来就被要求登录。
11. 头像下拉栏可以看到自己的好友号，并能直接打开添加好友弹窗。
12. 活动详情页增加“联系发起人”入口，可以直接进入与发起人的私聊。
13. 全站搜索支持查找好友号和昵称，搜索结果中可以直接发送好友申请。
14. 评论区支持编辑、删除和回复，活动前沟通更完整。
15. 用户可以举报用户、活动、组局和评论，举报入口文案更简洁。
16. 管理员后台可以查看和处理举报，并能区分待处理、已处理和驳回状态。
17. 评论被回复、活动收到新评论、举报提交和新举报待处理都会进入通知链路。
18. 好友、评论、私聊、分享、报名和组队转化等关键动作开始进入数据统计。
19. 活动发起人可以看到活动浏览、报名、沟通和分享等表现数据。
20. 管理员数据看台支持 7 天、30 天、90 天切换，并按总览、用户路径、来源价值和社区反馈分区查看。
21. 管理员数据看台增加趋势图、热门活动排行、公共活动来源价值和举报处理概览。
22. 微信内置浏览器登录 Google 时会显示引导页，提示用户用手机浏览器继续登录。
23. 页面加载时加入品牌 loading 动效，并根据当前语言显示加载文案。
24. 收藏能力扩展到活动和组队，用户可以更方便地标记感兴趣内容。
